# PKR v0.7 Persistent Intelligence

- Status: Public prerelease baseline
- Index: [PKR specification index](README.md)
- Scope: Memory, declarative Workflows, project Profiles, and Package lifecycle
- Depends on: [PKR v0.6 Steward and LPS Orchestration](0012-pkr-v0.6-steward-lps-orchestration.md)
- Successor: [PKR v0.8 Governed Evolution](0014-pkr-v0.8-governed-evolution.md)
- Publication: [`v0.7.0-alpha.1`](https://github.com/locooooooooo/PKR/releases/tag/v0.7.0-alpha.1)

## 1. Purpose

v0.7 makes governed work reusable across fresh sessions and different software
project types. It adds provenance-aware derived Memory, portable Workflow
guards, starter Profiles, and atomic Package lifecycle operations without
changing the core authority model.

## 2. Memory contract

Memory is a sequence-bound projection over authoritative records, events, and
Artifacts. Each entry MUST identify its sources and their revisions. Retrieval
MUST enforce project, public, principal, and Role visibility and MUST exclude a
source after supersession or loss of access.

Promoting a learned claim creates a governed object with provenance. A Memory
summary cannot override its sources, alter policy, or survive invalidated
provenance as current truth.

## 3. Workflow contract

Portable Workflows MAY use deterministic `eq`, `in`, `exists`, `and`, `or`, and
`not` expressions over supplied JSON context. Evaluation MUST NOT execute
arbitrary code, read time or network state, or add illegal core lifecycle
edges.

## 4. Profile and Package contract

- Profiles define project-specific Workflow and Verification behavior without
  kernel code changes.
- Package capabilities remain deny-by-default and MUST stay within the
  approved capability ceiling.
- Staging and activation are atomic; failed staging preserves the prior active
  set.
- Upgrade, uninstall, supersession, and rollback retain manifests,
  installations, provenance, and events.

The reference release includes distinct web and library Profiles to prove that
the same Runtime can host different governed project behavior.

## 5. Release and compatibility role

v0.7.0-alpha.1 is the first public Runtime-bearing prerelease. It is not a
stable compatibility contract and was never published to npm. The later v1
contract preserves its authority and rollback principles while superseding the
Alpha support and compatibility claims.

## 6. Non-goals

v0.7 does not provide semantic embedding search, cross-project Memory, a public
Package registry, autonomous policy changes, or binary Runtime self-update.
