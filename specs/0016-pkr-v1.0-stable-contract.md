## 1. Purpose

v1.0 is the first stable GitHub source release of the PKR framework and local
reference Runtime. It accepts one compatibility inventory while keeping
experimental integrations and evolution surfaces outside that promise.

## 2. Stable authority model

- SQLite state is authoritative; boards, projections, Memory indexes, and
  session locators are rebuildable views.
- Participant and Provider work reports are non-authoritative.
- Repository Verification is independent of the participant that performed
  the work.
- Runtime acceptance is a separate governed transition after Verification.
- Owner Decisions remain required for protected material changes.

## 3. Stable surface

The stable contract binds the listed CLI routes, root TypeScript exports,
versioned v0.2 and v0.4 schemas, persisted record kinds, ordered event types,
Verification plans, preflight reports, package root exports, support matrix,
and semantic-versioning rules. The
[machine inventory](../docs/release/v1-contract-manifest.json) is the exact
enumeration checked against source.

SQLite table layout, query plans, generated projection layout, private class
members, and unexported modules remain implementation details.

## 4. Experimental surface

Metrics, managed Prompt and Policy operations, governed evolution, optional
process Adapters, direct-store access, and related events are shipped for
evaluation but are outside the v1 stable compatibility promise. They cannot
weaken Repository Verification or Runtime acceptance.

## 5. Distribution and support

v1.0 supports Node.js 24.x, Python 3.11 conformance, Git, and the declared
Windows and Ubuntu CI environments. It is distributed through GitHub source
archives and checkouts. The npm package remains `private: true` and unpublished.

PKR is a trusted-host process, not an OS sandbox, cloud platform, hosted Agent
service, production SLA, or security certification. See the
[v1.0 release specification](../docs/release/v1.0.0.md) for its verification
and distribution evidence.
