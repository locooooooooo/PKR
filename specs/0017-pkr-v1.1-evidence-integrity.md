# PKR v1.1 Evidence Integrity and Sharing

- Status: Stable source release
- Index: [PKR specification index](README.md)
- Scope: Content-addressed RepositoryEvidence, migration, restore, and sharing
- Depends on: [PKR v1.0 Stable Contract](0016-pkr-v1.0-stable-contract.md)
- Extended by: [PKR v1.2 Supervisor and Runner](0018-pkr-v1.2-supervisor-runner.md)
- Release: [`v1.1.0`](https://github.com/locooooooooo/PKR/releases/tag/v1.1.0)

## 1. Purpose

v1.1 reduces repeated repository evidence while strengthening integrity,
restart, restore, and reviewed external sharing. It is a backward-compatible
minor extension of the accepted v1 contract.

## 2. RepositoryEvidence contract

RepositoryEvidence payloads are stored once per Project and content digest.
Sessions, Messages, Artifacts, commands, and external-effect journals retain
validated references rather than repeated diffs. Every reference MUST resolve
to payload bytes whose canonical length and digest match the authoritative
metadata.

Legacy inline evidence remains readable without rewriting historical record
revisions.

## 3. Migration and restore contract

- Migration from the v1.0 store format is transactional and retryable.
- Failure before commit leaves the prior schema and data authoritative.
- Checksummed snapshots include referenced evidence payloads.
- Restore verifies digest and canonical byte length before accepting data.
- Downgrade fails closed when the older format cannot preserve authoritative
  evidence references.

## 4. Sharing contract

`pkr projection export --profile shareable` produces an explicitly lossy,
redacted, size-bounded projection for review. It does not expose the local
projection as mutation input and does not imply that arbitrary project content
is safe to publish. Local Runtime state and evidence remain authoritative only
inside the governed project boundary.

## 5. Compatibility role

v1.1 adds one stable CLI route and five stable root TypeScript exports without
removing or reclassifying a v1.0 stable surface. The accepted semantic-versioning
and authority rules remain unchanged. See the
[v1.1 release specification](../docs/release/v1.1.0.md) and
[evidence operations guide](../docs/operations/evidence-and-projections.md).

## 6. Distribution boundary

v1.1 is a GitHub source release. The npm package remains private and unpublished.
Automated Git and Provider fixtures do not prove a live model service, external
Agent host, production workload, or adoption.
