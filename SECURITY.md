# Security policy

PKR is a local runtime. Do not put API keys, credentials, private prompts, or
production source diffs in issues or public audit files.

Report a suspected vulnerability privately through
[GitHub Security Advisories](https://github.com/locooooooooo/PKR/security/advisories/new)
before opening a public issue. Include the affected version, a minimal
reproduction, and whether the report involves persisted state, a provider
adapter, or the CLI.
