# PKR v1.2 release checklist

Run this checklist at one exact candidate SHA in a clean clone or independent
worktree. Do not continue after a failed command.

## Local candidate

```text
npm ci
python -m pip install --requirement conformance/requirements.txt
npm run verify
npm run check:contract
npm run check:version
npm run check:links
npm run check:sensitive
npm run check:public-tree
npm run check:package
npm pack --dry-run --json
npm run check:fresh-install
npm run audit:repository
npm run benchmark
npm run soak
npm run check:clean
```

- [ ] Package, lockfile, contract manifest, `VERSION`, source CLI, and packed
  CLI all report `1.2.0`.
- [ ] `package.json` remains `private: true`; no npm publish command or workflow
  runs.
- [ ] Supervisor coverage passes on the v1.1 store and RepositoryEvidence
  implementation for restart/replay, failed Verification, Lease recovery,
  protected decisions, concurrency fencing, and CLI watch.
- [ ] Every reconcile performs at most one controlled action, requires explicit
  Provider and verifier configuration, and stops at stable Owner attention for
  protected, missing, or ambiguous state.
- [ ] Provider reports and Skill metadata remain non-authoritative; only an
  independent Runtime Verification transition can accept work.
- [ ] Public-tree and packed-artifact scans contain no private development
  records, local state, credentials, absolute machine paths, or `demo/` data.
- [ ] Automated fake/local fixtures remain labeled and do not become live
  Provider, hosted-service, production, or adoption claims.

## Publication

- [ ] The exact public candidate SHA passes GitHub Actions on Windows and Ubuntu.
- [ ] Public `main` advances from v1.1.0 without rewriting published history.
- [ ] The exact public `main` SHA passes GitHub Actions before tagging.
- [ ] Annotated tag `v1.2.0` resolves to that green public `main` SHA.
- [ ] The non-prerelease GitHub Release links the same tag and records source
  distribution plus the npm non-publication boundary.
