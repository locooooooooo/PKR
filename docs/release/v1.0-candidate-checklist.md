# PKR v1.0 release checklist

Run this checklist in a clean clone or independent worktree at one proposed
candidate SHA. Record exit codes and artifact hashes in the
[release evidence template](release-evidence-template.md). Do not continue on a
failed command.

## Identity and clean state

```text
git rev-parse HEAD
git status --short
node --version
python --version
npm --version
```

- [ ] SHA is the candidate under review.
- [ ] Worktree is clean before the gate begins.
- [ ] Node is 24.x; OS is Windows or Ubuntu; Python is 3.11.

## Source and contract gates

```text
npm ci
py -3.11 -m pip install --requirement conformance/requirements.txt
npm run verify
npm run check:contract
npm run check:version
npm run check:links
npm run check:sensitive
node scripts/check-public-tree.mjs
npm run check:package
npm pack --dry-run --json
npm run check:fresh-install
npm run audit:repository
npm run benchmark
npm run soak
npm run check:clean
```

Use `python -m pip ...` on Ubuntu. The Windows launcher form avoids the Windows
Store `python` alias.

- [ ] Every command exits `0`.
- [ ] Generated files leave no unreviewed diff.
- [ ] The final clean-worktree check passes after every candidate command.
- [ ] Package, lockfile, contract manifest, `VERSION`, and CLI report `1.0.0`.
- [ ] `package.json` remains `private: true`; no npm publish command or workflow
  is run.
- [ ] Tarball and source smoke report the same package and CLI version.
- [ ] Public-tree inventory contains Runtime source and tests and excludes
  `iterations/`, local state, private remotes, credentials, and machine paths.
- [ ] Optional Adapter fixtures are labeled; no model/host compatibility claim
  is inferred.

## Release evidence

- [ ] The same SHA passes public CI on Windows and Ubuntu.
- [x] A non-author completed the documented newcomer path for the accepted
  pre-version candidate.
- [x] Reviewer `xjf` accepted G0-G9 and the zero-hidden-blocker declaration.
- [x] Reviewer `xjf` separately authorized G10 for the GitHub source release.
- [ ] Public source inventory, package inventory, checksums, and fresh-install
  logs are attached without secrets or private paths.
- [x] Owner signed the zero-hidden-blocker declaration.

After every item passes for the exact `1.0.0` commit, G10 permits merge to
public `main`, an annotated `v1.0.0` tag at the green main commit, and a
non-prerelease GitHub Release. It does not authorize npm publication.
