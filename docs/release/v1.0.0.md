# PKR v1.0.0

PKR v1.0.0 is the first stable source release of the AI-native project
framework and local reference Runtime. It gives humans, Agents, tools, and
workflows one governed project model without binding the framework to a model
vendor, Agent host, CLI, or execution service.

## Stable scope

- SQLite-authoritative project state for Mission, Goal, Task, Decision,
  Workflow, Agent, Workspace, evidence, events, and recovery;
- governed Steward intake and Agent-native claim, submit, and resume flows;
- non-authoritative participant work reports kept separate from independent
  Repository Verification and Runtime acceptance;
- provenance-aware Memory, declarative Workflows, project Profiles, and atomic
  Package lifecycle;
- source and local-tarball installation on Node.js 24 with Python 3.11 and Git;
- compatibility rules defined by the [v1 stable contract](v1-stable-contract.md).

Metrics, governed evolution, managed Prompts and Policies, direct-store
exports, and optional execution integrations are shipped as experimental
surfaces outside the v1 compatibility promise. Their evidence cannot bypass
Repository Verification or Runtime acceptance.

## Verification evidence

The release gates require:

- 53 of 53 Node tests plus schema, semantic, architecture, link, public-tree,
  package, license, dependency, and sensitive-data checks;
- a repository lifecycle audit covering successful completion, verification
  failure with durable `blocked` state, and restart recovery without silent
  acceptance or duplicate live work;
- source and packed-artifact fresh-install checks with matching `1.0.0` CLI and
  package versions;
- a bounded local benchmark and a 100-cycle, 1,000-seed-event restart soak;
- successful Windows and Ubuntu GitHub Actions jobs for the exact release
  candidate and final public `main` commit before the tag and Release are
  created;
- a [non-author newcomer trial](../audits/2026-07-22-newcomer-trial.md) of the
  documented Windows source-install and repository path.

Automated integration tests use bounded process fixtures. They do not prove a
live model service, external Agent host, production workload, or user adoption.

## Distribution and boundaries

Install from the GitHub source archive or a source checkout. The npm package is
`private: true`, is not published, and `npm publish` was not authorized for
this release.

PKR runs configured Verification commands and optional integrations as trusted
host processes with the current operating-system user's authority. It provides
policy, scopes, structured arguments, timeouts, evidence, and fail-closed state
transitions; it is not an OS sandbox, VM, container, credential vault, cloud
platform, hosted Agent marketplace, production SLA, or security certification.

See the [quickstart](../quickstart.md), [architecture](../architecture.md),
[operational limits](../operations/limits.md), and
[security policy](../../SECURITY.md) before using the Runtime on a repository.
