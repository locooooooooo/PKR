# PKR v1.1.0

PKR v1.1.0 is a backward-compatible source release focused on evidence size,
integrity, and safe projection. It preserves the accepted v1 authority model:
SQLite state is authoritative, participant work reports are non-authoritative,
Repository Verification is independent, and Runtime acceptance is separate.

## Changes

- RepositoryEvidence payloads are stored once per Project and content digest.
  Sessions, Messages, Artifacts, commands, and external-effect journals retain
  validated references instead of repeated Git diffs.
- Existing v1.0 `pkr.store/v1-candidate` databases migrate transactionally from
  SQLite `user_version=1` to `user_version=2`. Failure injection proves the
  schema change rolls back and can be retried.
- Checksummed snapshots include evidence payloads and verify their digest and
  canonical byte length during restore. Downgrade fails closed once the older
  format can no longer represent stored references.
- Legacy inline evidence remains readable without rewriting record revisions.
- `pkr projection export --profile shareable` writes a deliberately lossy,
  size-bounded export with explicit redaction notices. Local projections remain
  rebuildable views and are not mutation input.

The contract adds one stable CLI route and five stable root TypeScript exports.
It does not remove or reclassify a v1.0 stable surface.

## Verification boundary

Release gates cover fresh and migrated stores, transaction rollback, restart
replay, external-effect fencing, snapshot round trips, legacy compatibility,
shareable redaction, source and tarball installation, public-tree and sensitive
data checks, benchmark, soak, and hosted Windows/Ubuntu CI for the exact public
commit.

Automated Git and Provider fixtures do not prove a live model service, an
external Agent host, a production workload, or user adoption.

## Distribution

Install from the GitHub source archive or a source checkout. `package.json`
remains `private: true`; v1.1.0 is not published to npm and this release does
not authorize `npm publish`.
