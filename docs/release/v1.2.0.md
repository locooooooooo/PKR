# PKR v1.2.0

PKR v1.2.0 is a backward-compatible source release that adds an optional,
explicitly started Supervisor/Runner for one governed Task. It preserves the
accepted authority model: SQLite state is authoritative, Provider reports are
non-authoritative, Repository Verification is independent, and Runtime
acceptance is a separate transition.

## Changes

- `pkr supervise --once` performs at most one deterministic reconciliation
  action and returns a structured `pkr.supervisor-result/v1` result.
- `pkr supervise --watch` repeats that same composable step at an explicit
  interval until the Task is terminal or needs Owner attention.
- `pkr.supervisor/v1` configuration binds one existing Task, active Agent,
  explicit local Provider Adapter, independent verifier, Verification plan,
  and any required capabilities. PKR does not select a Provider automatically.
- Provider and Verification external effects are fenced in SQLite. Successful
  effects replay after restart; ambiguous pending effects stop for attention
  instead of running twice.
- Lease heartbeat, expiry, and one governed recovery dispatch compose existing
  Runtime/LPS behavior without creating a second authority source.
- Protected clarification, missing configuration, unresolved capability/Skill
  hints, blocked Tasks, and ambiguous effects fail closed with stable attention
  reasons. Skill metadata cannot grant identity, permission, or acceptance.

The experimental contract adds one CLI route, Supervisor root exports, two
versioned configuration/result contracts, and a narrow Runtime Verification
effect reservation API. No v1.1 stable surface is removed or reclassified.

## Verification boundary

Release gates cover Supervisor happy path and restart replay, failed independent
Verification, Lease lifecycle and recovery, protected/missing configuration,
concurrent Provider and Verification fencing, CLI once/watch behavior, the v1.1
RepositoryEvidence migration and projection paths, source and tarball installs,
public-tree and sensitive-data checks, benchmark, soak, and hosted
Windows/Ubuntu CI for the exact public commit.

Automated Git and fake/local Provider fixtures do not prove a live model
service, an external Agent host, a production workload, or user adoption. The
Supervisor is explicit local liveness infrastructure, not an always-on hosted
service or an OS sandbox.

## Distribution

Install from the GitHub source archive or a source checkout. `package.json`
remains `private: true`; v1.2.0 is not published to npm and this release does
not authorize `npm publish`.

See the [Runtime and Supervisor guide](../runtime-guide.md), the
